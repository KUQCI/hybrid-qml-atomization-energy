Quantum-Enhanced
Molecular Energy Prediction

Quantum layer first. Baselines second. One clear target to beat.

🎯 Project Goal

Build a hybrid quantum-classical model on QM7 that predicts atomization energies. Show the quantum layer contributes, then compare against classical baselines to tell a complete story for sponsors.

📏 Target Accuracy
< 1.0
kcal/mol MAE — the definition of chemical accuracy.
If we hit this, we're in legitimate research territory.

PHASE 01
Environment & Data Setup
~2–3 days
Load QM7 via DeepChem
Use dc.molnet.load_qm7() to load the dataset. DeepChem handles the download automatically. Inspect the returned DiskDataset: verify 7165 samples, confirm feature shape (Coulomb matrices) and label shape (atomization energies). Print min/max/mean of labels.
Person A
Split & preprocess via DeepChem
Use dc.splits.RandomSplitter (or ScaffoldSplitter) with frac_train=0.7, frac_valid=0.15, frac_test=0.15 and a fixed seed. Apply dc.feat.CoulombMatrix or use the built-in featurizer. Normalize labels with a dc.trans.NormalizationTransformer. Save the three split datasets so everyone uses the same ones.
Person A
Set up PennyLane + PyTorch environment
Install pennylane, pennylane-qiskit, torch. Run a 4-qubit smoke test circuit on default.qubit simulator. Confirm gradients flow through a TorchLayer. Write a one-file test script and commit it.
Person B
Set up repo & experiment logging
Create repo with /data, /models, /results, /notebooks. Add requirements.txt. Set up a lightweight logger that saves: model name, MAE (val + test), config (n_qubits, depth, lr, epochs) to a CSV. This keeps all runs comparable.
Person C
✅ Exit Criteria Data loads correctly on all machines. Smoke test circuit passes. Everyone has committed to the repo.
PHASE 02
Quantum Layer — Design & Build
~4–5 days
Design the variational ansatz (on paper first)
Decide: n_qubits (start with 4–6), encoding strategy (angle encoding of top-k eigenvalues), entanglement pattern (linear or all-to-all), number of variational layers. Document the design in a README before writing any code.
Person B
Implement the QNode in PennyLane
Build the circuit as a qml.QNode. Input: encoded feature vector. Output: expectation values of Pauli-Z on each qubit. Test it on 5 random samples. Confirm output shape is (n_qubits,).
Person B
Wrap as a differentiable PyTorch layer
Use qml.qnn.TorchLayer. Run a toy backward pass and confirm gradients are non-None. This is the gate: if gradients don't flow, the model can't train. Fix it before moving on.
Person B
Build the full hybrid model
Stack as a single nn.Module: (1) classical linear encoder → n_qubits dims, (2) TorchLayer (quantum), (3) linear output layer → scalar energy. The whole pipeline must be callable like a normal PyTorch model.
Person C
✅ Exit Criteria Hybrid model runs forward pass without error. Gradients flow end-to-end. Output is a scalar per sample.
PHASE 03
Training & Chasing Chemical Accuracy
~4–5 days
Build the training loop
MSE loss, Adam optimizer. Log train loss and val MAE every epoch. First run on 200 samples just to confirm loss decreases. Then scale to the full training set. Every run must be logged via the experiment CSV.
Person A
Hyperparameter tuning loop
Vary: learning rate (1e-3, 5e-4, 1e-4), number of epochs (50, 100, 200), circuit depth (1 vs 2 layers). Run each config, log all results. The goal is to get val MAE as close to 1.0 kcal/mol as possible.
Person A
Circuit architecture iterations
If MAE is stuck above 2.0 kcal/mol after tuning, try: more qubits, different encoding (IQP encoding vs angle), stronger entanglement. Each change is a new experiment entry in the log. Don't throw away old results.
Person B
Track best model & save checkpoints
Save torch.save() checkpoint for every run that improves val MAE. Keep the top-3 checkpoints. At end of phase, run the best checkpoint on the held-out test set and record the official test MAE.
Person C
⚠ Reality check: Quantum simulation on CPU is slow. Sub-sample to 500–1000 molecules if needed — this is standard in QML research and doesn't undermine the work. Prioritize getting clean results over using all 7165 samples.
✅ Exit Criteria You have a best test MAE from the hybrid model recorded. Even if it's above 1.0 kcal/mol — you have a number, and you know what you tried. That's enough to move forward.
PHASE 04
Classical Baselines & Comparison
~3 days
Implement KRR baseline
sklearn KernelRidge with Laplacian kernel on the same eigenvalue features, same train/test split. Grid search alpha and gamma. Target: reproduce ~3.1 kcal/mol from literature. Log result.
Person A
Implement classical-equivalent hybrid (ablation)
Replace the quantum TorchLayer with a classical linear layer of identical input/output dimensions. Train with identical settings as your best quantum run. This is the critical comparison: does the quantum layer actually help?
Person C
Optionally: try a quantum-enhanced SchNet-style model
If time permits and results are weak, explore inserting a quantum layer into a SchNet-like message passing network. This is the "quantum-enhanced SchNet" direction Osama mentioned. Not required for MVP, but worth attempting if Phase 3 MAE is disappointing.
Person B
Compile the results table
Build: Model | Input | Test MAE | vs Chemical Accuracy (<1.0). Rows: KRR baseline, classical-equivalent hybrid, quantum hybrid (best config), and optionally quantum-SchNet. This table is the centerpiece of your presentation.
All Three
✅ Exit Criteria Results table is complete. You have written 3–5 bullet conclusions: what worked, what didn't, and your hypothesis for why.
